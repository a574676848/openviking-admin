# OpenViking 知识库平台 - 前端 UI/UX 设计规范

## 1. 核心设计语言

### 1-1. 设计概念: Playful Engineering (趣味工程学)

打破 B 端系统的死板，引入高度互动和愉悦感的微交互系统。系统支持两套截然不同的**高明度视觉风格**，通过"主题切换"展现同一套数据的两种性格。
两套风格均围绕"极客感"、"高可读性"与"情绪交互"展开，摒弃原本的纯暗黑或传统蓝白面板。

### 1-2. 双色主题系统

- **主题 A：星智流光 (Luminous)**
  - **适用情绪**：智能、轻盈、温和的人文科技感。
  - **视觉特征**：极简清透的浅灰底色配合纯白卡片，去除硬阴影与粗黑边，采用**极细的边框（1px `#E3E3E3`）** 和**柔和多层的微投影（Soft Shadow: rgba(0,0,0,0.08)）**。
  - **颜色 (Color)**：全局以中性低饱和灰白为主，辅以清新的**草原绿（#16A34A）**作为焦点色，其余辅色也以柔和中性为主。
  - **边框与形状**：圆角放大至 `rounded-xl` (12px) 或全圆角 `rounded-full`，带来无攻击性的亲和力。交互反馈从剧烈下压变为柔和的背景色渐变和阴影加深（呼吸感）。

- **主题 B：Starry Sky (浩瀚星空暗黑风)**
  - **适用情绪**：高端、极客、深邃的数据掌控感。
  - **视觉特征**：极度克制的暗蓝/黑底色（`#0B0F19`），玻璃拟态半透明卡片，依靠微光阴影（Glow）和 1px 暗色细线划分层级。
  - **颜色 (Color)**：纯粹的深色层次。主文本为极具可读性的亮白（`#F8FAFC`）。全局使用高明度霓虹色点缀（如：电光蓝/发光青 `#00F0FF`），在暗黑中建立视觉锚点。
  - **边框与形状**：圆角放大至 `rounded-xl` (12px) 或全圆角 `rounded-full`，悬浮 (Hover) 会触发强烈的霓虹发光与轻微位移。

### 1-3. 排版系统

停用普通的 Arial 等默认字体，针对不同主题选用现代感极强的字体：

- **标题与正文 (Headings & Body)**：**Inter**，提供出色的屏幕可读性、干净的人文科技感与完美的国际化支持。
- **数字列/代码块 (Code & Stats)**：统一继承 **Inter + 中文无衬线字体栈**，确保控制台各区域的中文、数字与代码片段保持一致。

### 1-4. 空间、布局与图标

- **严禁 Emoji**：必须彻底杜绝任何原生 Emoji (😀/📁/⚙️)。所有图标强制使用 **Lucide React** 的几何型中性 SVG 图标 (`stroke-width="2"` 或 `1.5`)，通过 CSS 变量填充颜色（如纯黑或霓虹发光青色），维持硬核统一的视觉纯度。
- **激进的网格布局 (Radical Bento Grid)**：
  - 主内容区抛弃传统 Admin 的"松散堆叠"与"标准间距 (Gap)"。
  - 所有卡片采用 **Bento Grid (便当盒/蓝图网格)**。卡片之间极度贴合，犹如工程蓝图或拼图，用粗犷重线 (Light 模式) 或极细高亮线 (Dark 模式) 无缝分割区域，呈现极高密度的信息连接感。
- **解构侧边栏 (Floating Command Dock & Split-panel Terminal)**：
  - **弃用经典侧边栏**：不再使用传统的左侧贯穿式灰色死板菜单。
  - **Light 模式 (悬浮卡片 / 星智流光)**：左侧导航作为独立的卡片面板，悬浮 (Hover) 时通过极其细微的阴影变换和背景色微调（`var(--bg-elevated)`）提供反馈，去除突兀的物理位移。
  - **Dark 模式 (终端分屏控制台)**：左侧导航化作黑客终端的竖向分屏 (Split-panel)。当前激活的菜单前带有命令提示符 `>_`，并伴随闪烁的光标特效。整个界面就像切分良好的 Vim 或 Tmux 窗口。

### 1-5. 文本艺术与数据层级

- **痛点解法**：界面排版（Typography）本身就是设计的主体。
- **极致的数据比重**：核心数据和关键指标字号大幅夸大（例如 `text-6xl font-black`）。
- **信息对齐**：使用等宽字体 (Monospace) 对齐矩阵维度、Token Score 等微观数据。通过极致大小的反差和粗细（Black / Thin），在极简留白的区域建立最高视觉张力。

### 1-6. 文案边界：面向用户中文，面向协议英文

- **面向用户的主文案必须优先中文**：页面标题、副标题、按钮、空状态、错误提示、确认弹窗、成功反馈统一使用中文主表达，避免把英文协议词直接暴露成视觉主标题。
- **面向协议的字段保持英文准确性**：命令、API 路径、HTTP method、URI、Query、Rerank、TTL、API Key、SSE、session key 等协议术语保留英文，不做意译。
- **英文协议词只作为辅助信息出现**：当页面必须暴露协议细节时，优先采用“中文解释 + 英文术语”的写法，例如“复制客户端配置（SSE）”“最低分数（scoreThreshold）”。
- **错误信息要给出下一步**：CLI 和控制台的失败提示不能只返回原始错误码或英文 message，至少补一条中文修复建议，例如重新登录、检查租户权限、确认服务地址可访问。

## 2. 核心模块 UX 拆解与规范

### 2-1. 登录页 (/login)

- **布局**: 极简分割双栏或绝对居中。
- **交互**: 具备物理引擎弹回效果的输入框（Shake on Error）、双眼跟随光标的物理守望者组件（Viking Watchers）。
- **控制台风格**: 星智流光 / 浩瀚星空主题自由切换，Toast 弹窗摒弃原生，融入主题。

### 2-2. 核心骨架与控制台看板

- **结构**: **极简的顶部路由面包屑 (Header) + 蓝图式便当盒左侧路由 (Sidebar)**。
- **数字大屏**: 顶部巨型指标区块，纯粹基于数字自身的字形设计：
  - 接入知识库总数、Embedding 向量总数等，抛弃无意义的圆形色块和背景图。
- **时间线看板**: 将最近的知识录入、预处理、异常日志展示在时间线流上。

### 2-3. 知识树治理

- **左树右详情**:
  - **左侧**: 坚实致密的文件夹层级结构图表，Hover 突出操作项。
  - **右侧**: 高定制的数据表格 (Data Table) 或属性面板。无纵向分割线，只有极淡的横向细线。使用状态点 (红/绿/灰) 展示清洗、分段、切片和 Embedding 状态。

### 2-4. 交互式检索调优

- **问答调试区 (QA) 与检索调试区 (Search)**:
  - 置于顶部核心，借鉴终端指令输入感，支持多行文本 (Textarea)，带有 Cmd+Enter 快速提交提示。
  - 每一条召回知识切片 (Chunk) 单独成卡。卡片内重点用 Badge 标记：得分 (`Score: 0.895`)、所属文档库链路、分段序号。被匹配词元 (Tokens) 使用高饱和度荧光色背景高亮（如黄或青）。提供反馈（Feedback）交互锚点（👍 / 👎）。
- **检索设置 (Settings)**:
  - 参数控制面板，极度克制的表单。使用等宽字体展示 `top_k`, `score_threshold`, `rerank_enabled` 变量配置。左文右控结构。

### 2-5. 运营数据与分析

- **数据分析看板 (Analytics)**:
  - 多卡片 Grid 布局。
  - **图表设计**：禁止常规圆滑面积图。采用极细线条、直角方块或纯色叠加绘制趋势图和柱状图，展现高度精准的数据控制感。
  - 包含数据：检索命中率、检索耗时、总检索次数、URI 覆盖占比。
- **无答案分析 (Analysis)**:
  - 聚焦长尾请求的表格列表。高亮标识"零结果查询"，用醒目红色标签警示。提供一键跳转至导入知识库补充资源的捷径。

### 2-6. 运维与管控

- **系统监控 (System)**:
  - 高亮展示系统健康状况 (Health & Queue)。
  - 处理队列使用直观的警示色 (红 / 黄 / 绿) 定级，数字要求巨大且粗体，给人直接的冲击力和严谨感。
- **审计日志 (Audit)**:
  - 纯文字流的数据表格，包含时间戳、用户、操作行为 (Action)、IP 等。
  - 时间戳和 IP 强制使用等宽字体。操作行为采用各异的底色 Badge，迅速建立视觉索引。

### 2-7. 租户与用户管理

- **列表页**:
  - 紧凑且行距较小的管理列表，支持按角色/状态过滤。
  - 使用 Toggle 或超简单文案按钮控制启用 / 禁用（不用复杂的开关动画，采用方块切换或纯文本颜色变化）。
- **创建规则**:
  - 平台态创建用户时，租户绑定以表单选择为准。
  - 租户态创建用户时，租户绑定始终固定为当前租户。
  - 租户态不展示“租户选择”控件，避免与自动绑定规则冲突。
  - 提交前若缺少租户上下文，页面应阻止提交并给出中文错误提示。
- **详情与配额面板**:
  - 明确展现 `maxDocs`, `maxVectors` 等限制数据，使用极细进度条 (`height: 2px` 到 `4px`) 展示配额消耗。

### 2-8. WebDAV 接入面板

- **结构**:
  - 采用代码拷贝模式。左/上侧选择工具（Obsidian、VS Code、Windows、macOS），右/下侧展示带行号的 `json`、`ps1` 或 `sh` 代码块。
  - 连接信息区展示服务地址、用户名和 API key 来源说明，支持一键复制，但不回显完整 API key。
  - 连接自检通过同源代理调用服务端 `/console/webdav/check`，由服务端代发 `PROPFIND Depth: 0`，并将 `401`、`403`、`404`、`405` 等失败原因转成可读中文提示。
  - 页面文案要明确租户根目录可挂载全部有权限知识库，客户端首连在根目录执行 `MKCOL` 时会创建新的知识库，而不是要求用户手工填写某个知识库 ID。

## 4. 全局趣味交互与守望者系统

为了打破传统企业软件的死板，在 **星智流光 (Luminous)** 与极简风格的外壳下，融入**极具人情味和物理反馈的微交互系统**。
这套交互不再是卡通人物，而是化身为符合 AI 底座设定的"**维京守望者 (Viking Watchers)**"——它们可以是基于代码字符（ASCII 面孔）、极简的几何光效球体 (Neon Orbs) 或者机械阵列透镜，分布在系统的各个角落。

### 4-1. 登录页：好奇与隐私

- **输入账号时**：表单上方或侧边的"守望者"会紧紧跟着鼠标移动，当你聚焦到账号框时，它会微微"探头"或聚焦放大，表现出"看看你是谁"的好奇感。
- **输入密码时**：守望者会迅速闭上机械眼、拉下百叶窗或者完全背过身去（如果是多个节点，则整齐划一地转身），彻底切断对鼠标的跟随，传递出强烈的"非礼勿视、保护隐私"的安全感。

### 4-2. 检索问答页：思考与注视

- **输入检索问题时**："守望者"的视线会跟随你的光标在输入框内移动。
- **等待 AI 召回时**：守望者进入"检索状态"，眼睛变成脉冲光环 (Loading Spinner) 或飞速闪烁的代码符，表现出它正在知识库里翻找的物理过程。
- **召回结果时**：如果大获成功返回很多结果，它会对你"闪烁/点头"；如果查无结果，它可能会做出一个"垂头丧气"或者"光芒黯淡"的沮丧微动作。

### 4-3. 知识树与导入操作：警戒与扫描

- **知识树拖拽**：当你拖拽文件夹时，"守望者"会随着你拖拽的鼠标移动，仿佛在帮你盯着你要放去哪儿。
- **高危权限/设置修改 (如删除知识库、重索引)**：此时"守望者"的颜色会从正常的（冷蓝/白）瞬间切换成警报色（红/黄），并且眼神变得锐利 (Narrowing)，仿佛在说"你确定要这么做吗？"，进一步增加误操作前的防呆反馈。

### 4-4. 全局页面角落：陪伴

- 在整个后台的右下角或导航栏顶部，可以安放一个非常袖珍的"主节点"，它平时只是随意打量四周、偶尔眨眼缓解疲劳，让整个死寂的管理后台拥有了"生命力"。

> **研发落地备注**：技术上使用 `useMouse` 钩子捕获光标 `$x, $y`，通过基础的三角函数计算偏转角度（`Math.atan2`），赋予 SVG / CSS Transform 组件相应的 `rotate` 与 `translate` 值，配合高弹性的 `framer-motion` / `react-spring` 弹簧物理缓动动画，实现极其跟手且 Q 弹的视觉反馈。

## 5. 高级交互特效与赛博隐喻

为了进一步强化"底层知识引擎"的黑客与极客氛围，全站将穿插以下几个高级微交互，让冷酷的 B 端系统呈现出科幻电影般的质感：

### 5-1. 数据显影

- **适用场景**：总览看板的大数字 (KPI)、检索召回的高亮词、以及 AI 回答的逐字吐出时。
- **交互效果**：文本出现时不是普通的"淡入 (Fade-in)"，而是以一串随机的乱码（如 `0xA4`, `!@#$`, `1010` 等十六进制或特殊符号）高速闪烁，接着在半秒内迅速锁定解码成真实的数字或文字。这种"黑客破译"的显影感能极大增强界面的专业性和神秘感。

### 5-2. 边缘脉冲电流

- **适用场景**：搜索等待状态、导入任务进行中。
- **交互效果**：抛弃传统的中间转圈加载动画 (Spinner)。当系统在思考或处理巨量数据时，沿着卡片极其锋利的 `1px` 边框，会有一道高亮的霓虹细线（蓝/绿）像电流般飞速环绕奔跑，仿佛数据正在芯片线路中极速穿梭。

### 5-3. 知识向量空间与全息生态背景

- **适用场景**：全局登录页、以及登录后的所有业务页面底板。
- **交互效果**：两套主题拥有截然不同但互相呼应的全息底板生态。
  - **浩瀚星空 (Starry Sky)**：背景采用深邃的粒子流明网格。偶尔会有**随机斜向流星雨 (Meteor Shower)** 从右上至左下以极其悠长、克制的速度划过，流星头部带有巨大的加色发光晕，能真实照亮界面背后的玻璃拟态卡片。
  - **星智流光 (Luminous)**：背景采用极简圆点网格。底层游荡着一群极具生机的**“数据萤火虫” (Data Fireflies)**。它们拖着黄绿色的发光尾部、高频扇动着半透明的翅膀，在网格中以物理惯性弹跳、游走，并在飞出屏幕后随机繁衍诞生，形成一个具有呼吸感的智能生态。

### 5-4. 守望者视线追踪系统 (Watcher Gaze Tracking)

- **适用场景**：登录页核心、系统防呆提示。
- **交互效果**：
  - **常规模式**：Viking Watcher 会通过 `useSpring` 极度平滑地追踪用户鼠标的移动轨迹。
  - **生态接管模式**：当用户在界面上**停止操作鼠标超过 2 秒**时，Watcher 的视线会智能接管。在“浩瀚星空”下，它会温柔地抬头凝视划过的流星；在“星智流光”下，它会好奇地跟踪距离它最近的一只数据萤火虫的飞行轨迹，展现出系统静谧时的自我意识。
  - **隐私模式**：输入密码时，Watcher 会立刻闭上双眼切断一切视线追踪。

### 5-5. 坍缩碎裂删除

- **适用场景**：删除错误知识库、删掉多余节点。
- **交互效果**：B端系统删除操作一般压力很大，可以通过强烈的视觉反馈释放压力：需要删除的行或者对象在确认删除后，瞬间"花屏 (Glitch)"或者像受到重力一般被挤压成一条线最后消失，伴随轻微的屏幕震动感，极其解压。

### 5-6. 隐藏的终端彩蛋模式

- **适用场景**：全局任意顶栏的搜索框/输入框。
- **交互效果**：如果你敲入特定的指令如 `/sudo override` 或 `/godmode` 后回车，整个原先深灰色相间的界面会瞬间全屏黑化，所有字体变成粗旷的荧光绿等宽字符（CRT 磷光管显示器风格），并在几秒内滚过满屏の极客加载代码，进入一个隐藏的彩蛋页或者专属的极客操控台。

## 6. 控制台交互规范

控制台页面需要在高密度数据、频繁操作和长期使用场景下保持稳定、克制和可扫描。以下规范用于统一页面形状、动画、操作分级和滚动行为。

### 6-1. 标准化形状系统 (Radius System)

不再使用硬编码的 Tailwind 圆角类，统一通过全局 CSS 变量控制，以实现主题间的瞬间切换：

- **`--radius-base`**: 容器与主卡片的圆角。星智流光为 `28px`，浩瀚星空为 `28px`。
- **`--radius-pill`**: 交互按钮与状态标签的圆角。星智流光为 `9999px` (Pill)，浩瀚星空为 `9999px`。
- **`--radius-tile`**: 内部小组件（如输入框、搜索条）的圆角。星智流光为 `16px`，浩瀚星空为 `16px`。

### 6-2. 主题感知动画策略 (Animation Policy)

为了响应用户对极致静谧感的需求，引入了动态动画控制：

- **变量控制**: 通过 `--pulse-animation` 变量定义呼吸效果。
- **静止原则**: 在特定需求下，该变量可设为 `none`，但星空主题中支持星芒闪烁与呼吸动画。
- **实现**: 统一使用 `.animate-theme-pulse` 类替代 Tailwind 原生的 `animate-pulse`。

### 6-3. 表格操作分级模式 (Action Tiers)

针对管理列表（租户、用户等）按钮过多导致的“错行”与“视觉杂乱”问题，确立了按钮分级规范：

- **一级操作 (Primary Actions)**: 仅保留 1-2 个最高频或核心动作（如“进入空间”、“状态切换”）。
- **二级操作 (Secondary Actions)**: 使用 `PlatformActionMenu` 组件进行折叠。
- **更多操作 (The More Button)**: 折叠触发器必须包含“更多”文字标签，提升识别度。菜单必须使用 `React Portal` 渲染至全局，以防被表格容器的 `overflow: hidden` 剪裁。

### 6-4. 隐形式滚动条 (Invisible Scrollbars)

- **视觉规范**: 全局弹窗与长列表默认隐藏滚动条轨道。
- **交互逻辑**: 仅在鼠标悬浮 (Hover) 时显示极细的（4px）浅色滑块，滑块圆角需符合主题规范（Neo 为 Pill，Swiss 为直角）。

### 6-5. 关键操作可见性

- **悬浮反馈**: 在浅色表格行背景下，默认状态的按钮应具备 `border` 或 `shadow`，在 Hover 时需触发 `var(--brand)` 品牌色描边。
- **删除确认**: 所有删除动作必须收纳至二级菜单，并明确标记为 `tone="danger"`（文字变红且 Hover 时带淡红背景）。

## 3. 技术栈对应与落地约束

- **CSS 框架**: Tailwind CSS
- **组件库映射**: 基于 `shadcn/ui`，但在使用前需配置 `components.json`：
  - Style: `Default` / `New York`
  - 将其内部全局 `radius` 设置为 `0` 或是 `0.25rem`。
- **图标系统**: `Lucide React` (线条图标，统一设置 `strokeWidth={1.5}`)
- **中英文字体**: 载入 `next/font/google` 方案调用 Inter 家族，并通过 `font-sans` 统一继承 `PingFang SC`、`Microsoft YaHei` 等中文无衬线 fallback。
- **深色模式**: `next-themes` 实现 `dark` class 的无缝切换。

> ⚠️ **落地原则要求**：
> 开发工程师在实现时，如果功能可以用边框、文字排版、字号区分，就坚决不用填充色；如果能用直线，就禁止用曲线。保持最高程度的设计纪律。
