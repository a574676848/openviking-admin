# ADR 0008: 前端设计组件层与主题系统约束

## 状态

拟采纳

## 背景

`docs/DESIGN.md` 已经定义了主题语言，但页面中仍有较多局部 Tailwind class 和页面级拼装逻辑，导致移动端导航、可访问性和中文友好化难以系统收敛。

## 决策

- 控制台 UI 逐步沉淀为组件层，页面优先消费语义组件而不是直接堆叠样式。
- 移动端导航、icon button、tooltip、focus ring 和空状态采用统一约束。
- 中文面向用户，英文保留给协议、命令、字段和调试上下文。

## 取舍

- 优点：视觉一致性、无障碍和多主题切换更可控。
- 成本：短期内需要重构若干页面布局和局部 class。

## 后续影响

- 新页面默认需要满足移动端导航与 `aria-label` 基线。
- 视觉回归测试应覆盖主题切换与关键页面文本溢出。
